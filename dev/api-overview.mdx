title: "Nexu controller API overview"

---

The Nexu controller runs a local HTTP server that the web dashboard uses for all configuration and runtime operations. You can also call this API directly — for example, to automate bot creation, integrate with external tooling, or build custom dashboards. Every route is typed end-to-end from Zod schemas through to a generated TypeScript SDK, so the spec always reflects the live code.

## Base URL

```
http://localhost:<port>/api/v1
```

The port is configured in `apps/controller/.env`. Check `CONTROLLER_PORT` (or `runtime-ports.json` when the stack is managed by the desktop launcher).

## Authentication

All requests require a bearer token. Pass the value of `GATEWAY_TOKEN` from the controller environment in the `Authorization` header:

```bash
Authorization: Bearer <GATEWAY_TOKEN>
```

<Info>
  When using the Nexu desktop app, the token is managed automatically by the launcher. You only need to supply it manually when calling the API from external scripts or tooling.
</Info>

## OpenAPI spec

The controller generates an OpenAPI 3.1 document at runtime. Two paths expose it:

| Path | Description |
|---|---|
| `GET /api/v1/openapi.json` | Machine-readable JSON spec |
| `GET /api/v1/doc` | Interactive Swagger UI |

The spec is auto-generated from Zod schemas via `@hono/zod-openapi`. After modifying any route or schema, regenerate the frontend SDK:

```bash
pnpm generate-types
```

## Resource groups

| Group | Description | Reference |
|---|---|---|
| **Bots** | Create, read, update, delete, pause, and resume AI agents | [Bots API](/dev/api-bots) |
| **Channels** | Connect and manage messaging platform integrations | [Channels API](/dev/api-channels) |
| **Models** | List available models and configure BYOK providers | [Models API](/dev/api-models) |
| **SkillHub** | Browse the skill catalog, install, and uninstall skills | [SkillHub API](/dev/api-skills) |

## Error responses

All error responses return JSON with at minimum a `message` field:

```json
{
  "message": "Bot not found"
}
```

Channel connect errors return a richer payload (see [Channels API](/dev/api-channels) for details).

<Note>
  The frontend web app must always use the generated SDK at `apps/web/lib/api/` and must not call controller routes with raw `fetch`. The SDK is regenerated by `pnpm generate-types` whenever routes or schemas change.
</Note>
